痛点 vs 解法

! 当前痛点
  • 肉眼经验依赖个人技术，新手难掌握
  • 化学检测成本高、周期长，且破坏样品
  • 现有工具只给结论，不告诉"为什么"和"怎么办"
✓ Fish-Agent 解法
  • 鱼眼照片入口，低成本非破坏式
  • FishFreshNetV1（EfficientNet-B0 + CBAM）三分类，准确率 99.23%
  • Grad-CAM 热力图解释"为什么"，结构化报告给出"怎么办"
  • 置信度驱动自适应分支，低置信度主动提示重拍

16 节点职责明细

#节点类型职责
1image_uploadtask接收图像并记录上传时间
2enhanced_quality_checkagent判断是否鱼类、是否含鱼眼、质量是否合格
3quality_unqualifiedtask输出不合格原因和重拍建议
4image_preprocesstask按场景和质量等级做轻量预处理
5fish_region_detectionagent识别鱼眼、鳃部、体表区域
6freshness_classificationagent优先 FishFreshNetV1，失败回退多模态大模型
7multi_region_fusionagent中置信度路径，融合鱼眼/鳃部/体表
8backup_model_reviewagent低置信度路径，大模型复核仲裁
9low_confidence_prompttask二次低置信度提示重拍或人工确认
10enhanced_gradcamagent调真实 Grad-CAM，失败生成降级解释
11structured_analysisagent鱼眼外观/清澈度/颜色/纹理/质量指标
12temporal_analysisagent新鲜度趋势与剩余保质时间预测
13report_generationagent结构化报告与知识图谱匹配
14recommendation_generationagent储存/食用/处理/安全/最佳实践建议
15result_visualizationtask汇总最终展示数据
16feedback_looptask生成记录 ID，预留样本池入口
agent 节点（调用大模型） task 节点（纯数据处理）

创作历程

💡

灵感诞生

陪家人去菜市场买鱼，摊主信誓旦旦"绝对新鲜"，回家切开却已发臭。发现鱼眼是新鲜度最强的视觉信号，但人眼无法分辨细微差距——如果 AI 能看懂鱼眼呢？

🔬

独立研究

撰写论文《FishFreshNetV1: A Lightweight and Explainable Framework》，提出 EfficientNet-B0 + CBAM 架构。构建 MFED 数据集（4800 张 / 4 环境 / GB 2733-2015 国标），达成 99.23% 准确率。数据集开源至 Mendeley Data。

⚙️

后端开发

FastAPI + PyTorch 模型推理服务，部署至 Hugging Face Spaces。LangGraph 16 节点智能体工作流，置信度自适应分支 + 多模态 LLM 兜底，确保每一次检测都有可靠结果。

🎨

前端开发（初赛）

TRAE Work 5 次对话从零到交付。项目奠基 → 视觉升级 → API 集成 → 结果深化 → 打磨交付。单文件 HTML 149KB，深海动态主题 + 4-Tab 分析报告 + 离线演示模式。

🚀

复赛迭代

多页面重构 → 移动端深度适配 + PWA → 16 节点工作流融合 → 置信度自适应前端机制 → 最终交付打磨。从"能用的 Demo"进化为"好用的产品"。
